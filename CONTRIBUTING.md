# Contributing to Encryptum

Thank you for considering contributing to Encryptum! We welcome contributions from everyone, whether it's through code, documentation, bug reports, or other helpful contributions.

## How to Contribute

1. **Fork the Repository**
   - Fork the Encryptum repository to your GitHub account and clone it to your local machine.

2. **Create a Feature Branch**
   - Create a new branch for your feature (e.g., `git checkout -b feature/my-feature`).

3. **Make Your Changes**
   - Make sure your changes adhere to the existing code style and project structure. 
   - Update documentation as necessary.

4. **Test Your Changes**
   - Thoroughly test your changes to ensure stability and quality.

5. **Commit Your Changes**
   - Write clear, concise commit messages explaining your changes.
   - Example: `Fix bug in memory synchronization`.

6. **Push to Your Fork**
   - Push your changes to your forked repository on GitHub.

7. **Submit a Pull Request**
   - Open a pull request (PR) to the main repository.
   - Provide a detailed description of the changes you made and the problem they solve.

## Code of Conduct

By contributing to this project, you agree to follow the [Code of Conduct](./CODE_OF_CONDUCT.md).

## Reporting Bugs or Issues

If you find a bug, please open an issue in the [Issues section](https://github.com/encryptumdev/encryptum/issues).

## Thank You!

We are excited to have you as a contributor and appreciate your efforts to make Encryptum better!
